## ✨ Features

<br>

### 🧠 1. Advanced AI Extraction — Gemini 2.5 Flash

The AI reads your content deeply and produces fully organized exam questions — no matter the input format.

```
 📝 Syllabus Text   ─┐
 📄 PDF Documents   ─┼──►  🤖 Gemini 2.5 Flash  ──►  ☑ MCQs
 🖼️  Images/Photos  ─┤                              ☑ Short Questions
 📋 Scanned Papers  ─┘                              ☑ Long Questions
```

- **Deep Content Analysis** — Understands syllabus text, book pages, notes, and full PDF documents
- **Smart Auto-Categorization** — Separates content into MCQs, Short Questions, and Long Questions automatically
- **Anti-Laziness Prompting** — Strict prompt engineering ensures every file is processed thoroughly with no skipping or hallucination
- **Existing Paper Detection** — Upload a scanned exam paper and the AI extracts questions exactly as-is

<br>

### 🌍 2. Perfect Bilingual Formatting — Invisible Grid System

```
 ┌──────────────────────────────┬──────────────────────────────┐
 │ English (Left)               │             (دائیں) اردو        │
 ├──────────────────────────────┼──────────────────────────────┤
 │ Q1. What is photosynthesis?  │  س١. ضیاء ترکیب کیا ہے؟         │
 │ a) A process of respiration  │  الف) سانس کا عمل               │
 │ b) Food making by plants     │  ب) پودوں کا کھانا بنانا            │
 └──────────────────────────────┴──────────────────────────────┘
            Zero-border invisible table — always aligned
```

- **Side-by-Side Layout** — English and Urdu (Nastaleeq) rendered perfectly in one document
- **Invisible Grid System** — Zero-border tables keep columns locked — never misaligned
- **Board Standard Compliant** — Matches Punjab Board and Federal Board formatting exactly
- **Jameel Noori Nastaleeq** — Correct RTL rendering with proper Urdu typography

<br>

### ⚙️ 7. Bulletproof Backend Architecture

```
 Request ──► FastAPI ──► Gemini AI ──► doc_generator ──► .docx ✅
               │
               ├── HTTP 429?  ──►  Switch to next API Key
               ├── Key 1 done ──►  Try Key 2
               ├── Key 2 done ──►  Try Key 3
               └── All done   ──►  Frontend countdown timer
```

- **Multi-Key Rotation** — Up to 3 Gemini API keys; quota exhausted on one? Next key activates instantly
- **Auto-Retry** — HTTP 429 handled gracefully with frontend countdown — no crashes, no page reloads
- **Unique Filenames** — Timestamp-based names prevent overwrites

## 🚀 Getting Started

### Prerequisites

- ✅ [Python 3.8+](https://python.org)
- ✅ [Gemini API Keys](https://aistudio.google.com/) — free from Google AI Studio
- ✅ [VS Code](https://code.visualstudio.com/) with Live Server extension
- ✅ [Git](https://git-scm.com/)

<br>

### Step 1 — Clone

```bash
git clone https://github.com/Shannaseem/AI-Test-Generator.git
cd AI-Test-Generator
```

### Step 2 — Virtual Environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

> 💡 You'll see `(venv)` in your terminal — that means it's active.

### Step 3 — Install Dependencies

```bash
pip install -r backend/requirements.txt
```

### Step 4 — Configure API Keys

Create `backend/.env` and add your keys:

```env
GEMINI_API_KEY_1=your_first_key_here
GEMINI_API_KEY_2=your_second_key_here
GEMINI_API_KEY_3=your_third_key_here
```

> ⚠️ This file is in `.gitignore` — it will **never** be committed. Keep it private.

### Step 5 — Start Backend

```bash
cd backend
python -m uvicorn main:app --reload --host 0.0.0.0 --port 10000
```

Expected output:

```
INFO:  Uvicorn running on http://0.0.0.0:10000
INFO:  Application startup complete.
```

### Step 6 — Launch Frontend

1. Open `frontend/` in VS Code
2. Right-click `index.html` → **Open with Live Server**
3. Browser opens — you're ready! 🎉

## 🔌 API Reference

**Base URL:** `https://ai-test-generator-2hsf.onrender.com` &nbsp;|&nbsp; Local: `http://localhost:10000`

<br>

#### `GET /`

```json
{ "status": "ok", "message": "API is running perfectly!" }
```

#### `POST /process-test`

Main generation endpoint — accepts `multipart/form-data`.

| Parameter        | Type   | Required | Description                |
| ---------------- | ------ | :------: | -------------------------- |
| `academy_name`   | string |    ✅    | School or academy name     |
| `subject`        | string |    ✅    | Subject name               |
| `class_name`     | string |    ✅    | Class or grade             |
| `test_date`      | string |    ✅    | Format: `YYYY-MM-DD`       |
| `time_allowed`   | string |    ✅    | Duration in minutes        |
| `syllabus`       | string |    ✅    | Chapter or topic           |
| `template_style` | string |    ✅    | `"table"` or `"column"`    |
| `bilingual`      | string |    ✅    | `"yes"` or `"no"`          |
| `exam_pattern`   | string |    ✅    | `"chapter"` or `"board"`   |
| `short_groups`   | string |    ✅    | `"1"`, `"2"`, or `"3"`     |
| `short_total`    | string |    ✅    | Total short questions      |
| `short_attempt`  | string |    ✅    | Short questions to attempt |
| `long_total`     | string |    ✅    | Total long questions       |
| `long_attempt`   | string |    ✅    | Long questions to attempt  |
| `long_q_marks`   | string |    ✅    | e.g. `"9"` or `"4+5"`      |
| `magic_prompt`   | string |    ❌    | Custom AI instruction      |
| `text`           | string |    ❌    | Syllabus text              |
| `files`          | file[] |    ❌    | PDFs or images (max 10)    |

```json
{
  "docx_filename": "Class_9_Physics_1748291234.docx",
  "ai_data": { "mcqs": [...], "short_qs": [...], "long_qs": [...] }
}
```

#### `POST /refine-test`

Same as `/process-test` plus:

| Parameter       | Type   | Required | Description         |
| --------------- | ------ | :------: | ------------------- |
| `ai_data_json`  | string |    ✅    | Current test JSON   |
| `refine_prompt` | string |    ✅    | AI edit instruction |

#### `GET /get-file/{filename}`

Returns the generated `.docx` file as a download.

## 📁 Project Structure

```
AI-Test-Generator/
│
├── backend/
│   ├── main.py                ← API endpoints, CORS, error handling, file routing
│   ├── ai_service.py          ← Gemini AI: extraction, refinement, key rotation
│   ├── doc_generator.py       ← Word engine: bilingual grid, headers, all sections
│   ├── requirements.txt       ← Python dependencies
│   └── .env                   ← 🔒 API keys — git-ignored, never commit
│
├── frontend/
│   ├── index.html             ← Dashboard layout, upload zone, Word preview modal
│   ├── style.css              ← Full UI: glassmorphism, progress bar, responsive
│   └── script.js              ← Logic: API calls, drag-drop, cache-busting, modal
│
├── .gitignore                 ← Excludes .env, __pycache__, venv, generated files
└── README.md                  ← This file
```

**Key files explained:**

| File               | Responsibility                                                                       |
| ------------------ | ------------------------------------------------------------------------------------ |
| `main.py`          | Defines all 4 API routes, handles CORS, generates unique filenames                   |
| `ai_service.py`    | Builds the Gemini prompt, parses JSON response, rotates API keys on 429              |
| `doc_generator.py` | Creates the entire `.docx` — page setup, header, MCQ tables, bilingual grid, marks   |
| `index.html`       | The full single-page app UI with modal viewer                                        |
| `script.js`        | Collects form data, calls APIs, simulates progress bar, manages Office Online viewer |
