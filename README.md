## 🌟 Why AI Test Generator PRO?

Creating a professional exam paper manually takes hours — formatting, balancing marks, writing questions, ensuring board standards. **AI Test Generator PRO eliminates all of that.**

```
  BEFORE  ──────────────────────────────────────────────────────────

  📚 Read syllabus → ✍️ Write questions manually → 📐 Format in Word
  → 🔁 Repeat for each section → ⏰ 2–3 hours of work

  AFTER   ──────────────────────────────────────────────────────────

  📤 Upload syllabus → ⚡ Click Generate → 📄 Download Word file
  → ✅ Done in under 60 seconds
```

## ✨ Features

<br>

### 🧠 1. Advanced AI Extraction — Gemini 2.5 Flash

```
  Input anything  →  AI extracts everything  →  Organized output
  ─────────────────────────────────────────────────────────────
  📝 Syllabus Text      ──►   ☑ MCQs (Multiple Choice)
  📄 PDF Documents      ──►   ☑ Short Questions
  🖼️  Images / Photos   ──►   ☑ Long Questions
  📋 Exam Paper Scans   ──►   ☑ Parts (a) & (b) for Board
```

- **Deep Content Analysis** — Reads and understands syllabus text, book pages, handwritten notes, and entire PDF documents
- **Smart Auto-Categorization** — Intelligently separates content into MCQs, Short Questions, and Long Questions with correct difficulty levels
- **Anti-Laziness Prompting** — Custom strict prompt engineering forces the AI to process every single uploaded file thoroughly — no skipping, no hallucinating extra questions
- **Existing Paper Detection** — If you upload an already-made exam paper (image or PDF), the AI extracts its questions exactly as-is instead of inventing new ones

### 🌍 2. Perfect Bilingual Formatting — Invisible Grid System

```
  ┌─────────────────────────────┬─────────────────────────────┐
  │  English (Left Side)        │         اردو (دائیں طرف)   │
  ├─────────────────────────────┼─────────────────────────────┤
  │  Q1. What is photosynthesis │  س١. ضیاء ترکیب کیا ہے؟   │
```

- **Side-by-Side Excellence** — English and Urdu (Nastaleeq script) rendered simultaneously in the same document

- **Board Standard Compliant** — Formatting matches official Punjab Board and Federal Board paper layout standards exactly
- **Jameel Noori Nastaleeq Font** — Correct RTL (right-to-left) text rendering with proper Urdu typography

### ⚙️ 7. Bulletproof Backend Architecture

```
  Request ──► FastAPI ──► Gemini AI ──► doc_generator ──► .docx
                │                                           │
                ├── Rate Limit Hit? ──► Switch API Key      │
                ├── Key 1 exhausted? ──► Try Key 2          │
                ├── Key 2 exhausted? ──► Try Key 3          │
                └── All exhausted? ──► Countdown timer      │
                                                           ▼
                                               ✅ File ready for download
```

- **Multi-API Key Rotation** — Automatically cycles through up to 3 Gemini API keys; if one quota runs out, the next key is used instantly without any interruption
- **Auto-Retry on Rate Limits** — Handles HTTP 429 errors gracefully with intelligent retry logic and a frontend countdown timer so the user knows exactly when to expect results
- **Unique Filename Generation** — Every generated file gets a unique timestamp-based name preventing overwrites and collisions
- **CORS Enabled** — Fully configured for cross-origin requests, ready for any frontend deployment

## 🚀 Getting Started

### ✅ Prerequisites

Before you begin, make sure you have:

- **Python 3.8+** installed → [Download here](https://python.org)
- **Google Gemini API Key(s)** → [Get free keys from Google AI Studio](https://aistudio.google.com/)
- **VS Code** with Live Server extension → [Download here](https://code.visualstudio.com/)
- **Git** installed → [Download here](https://git-scm.com/)

<br>

### Step 1 — Clone the Repository

```bash
git clone https://github.com/Shannaseem/AI-Test-Generator.git
cd AI-Test-Generator
```

<br>

### Step 2 — Create a Virtual Environment

```bash
# Create the virtual environment
python -m venv venv

# Activate it — Windows:
venv\Scripts\activate

# Activate it — Mac / Linux:
source venv/bin/activate
```

> 💡 You should see `(venv)` at the start of your terminal line — that means it's active.

<br>

### Step 3 — Install Dependencies

```bash
pip install -r backend/requirements.txt
```

<br>

### Step 4 — Configure API Keys

Create a file named `.env` inside the `backend/` folder:

```
backend/
└── .env         ← Create this file
```

Paste your Gemini API keys inside it:

```env
GEMINI_API_KEY_1=your_first_key_here
GEMINI_API_KEY_2=your_second_key_here
GEMINI_API_KEY_3=your_third_key_here
```

> ⚠️ **Important:** This file is listed in `.gitignore` and will never be committed. Never share it publicly.

<br>

### Step 5 — Start the Backend Server

```bash
cd backend
python -m uvicorn main:app --reload --host 0.0.0.0 --port 10000
```

You should see:

```
INFO:     Uvicorn running on http://0.0.0.0:10000 (Press CTRL+C to quit)
INFO:     Started reloader process using StatReload
INFO:     Started server process
INFO:     Application startup complete.
```

> ✅ Your backend is live at `http://localhost:10000`

<br>

### Step 6 — Launch the Frontend

1. Open the `frontend/` folder in **VS Code**
2. Right-click `index.html` → **"Open with Live Server"**
3. Your browser opens automatically — you're ready! 🎉

## 💡 How to Use

```
  ╔══════╗   ╔══════╗   ╔══════╗   ╔══════╗   ╔══════╗   ╔══════╗
  ║  1   ║──►║  2   ║──►║  3   ║──►║  4   ║──►║  5   ║──►║  6   ║
  ║Config║   ║Logic ║   ║Upload║   ║  AI  ║   ║Review║   ║Print ║
  ╚══════╝   ╚══════╝   ╚══════╝   ╚══════╝   ╚══════╝   ╚══════╝
```

**Step 1 — Configure the Paper**
Fill in Academy Name, Subject, Class/Grade, Date, Time Allowed, and Syllabus. Select Document Style (Table or Two-Column) and Language (English Only or Bilingual English+Urdu).

**Step 2 — Set Question Logic**
Choose your Exam Pattern (Chapter Test or Board Pattern). Set the exact number of MCQs, Short Questions (total and attempt), and Long Questions (total, attempt, and marks per question).

**Step 3 — Add Source Material**
Paste syllabus text directly, drag-and-drop PDF/image files, or press `Ctrl+V` to paste a screenshot. You can combine text AND files together for richer extraction.

**Step 4 — Generate**
Click **"Build Interactive Test Paper"**. Watch the animated progress bar go through: _Extracting AI Data → Drafting Questions → Formatting Word Document → Finalizing Layout → 100%!_

**Step 5 — Review & Refine**
A full-screen MS Word preview opens automatically. If anything needs changing, type a command in the **AI Command Box** at the bottom (e.g., _"Replace Short Q5 with a definition question"_) and click **Update Test**.

**Step 6 — Download & Print**
Click **"Download Exact Word File"** — your perfectly formatted, print-ready exam paper is saved to your computer.

## 🔌 API Reference

Base URL: `https://ai-test-generator-2hsf.onrender.com` (Production) or `http://localhost:10000` (Local)

<br>

### `GET /`

Health check endpoint.

```json
Response: { "status": "ok", "message": "API is running perfectly!" }
```

<br>

### `POST /process-test`

Main generation endpoint. Accepts `multipart/form-data`.

| Parameter        | Type   | Required | Description                              |
| ---------------- | ------ | -------- | ---------------------------------------- |
| `academy_name`   | string | ✅       | Name of the school/academy               |
| `subject`        | string | ✅       | Subject name                             |
| `class_name`     | string | ✅       | Class or grade                           |
| `test_date`      | string | ✅       | Date in `YYYY-MM-DD` format              |
| `time_allowed`   | string | ✅       | Time in minutes                          |
| `syllabus`       | string | ✅       | Chapter or topic name                    |
| `template_style` | string | ✅       | `"table"` or `"column"`                  |
| `bilingual`      | string | ✅       | `"yes"` or `"no"`                        |
| `exam_pattern`   | string | ✅       | `"chapter"` or `"board"`                 |
| `short_groups`   | string | ✅       | `"1"`, `"2"`, or `"3"`                   |
| `short_total`    | string | ✅       | Total short questions                    |
| `short_attempt`  | string | ✅       | Short questions to attempt               |
| `long_total`     | string | ✅       | Total long questions                     |
| `long_attempt`   | string | ✅       | Long questions to attempt                |
| `long_q_marks`   | string | ✅       | Marks per long Q (e.g. `"9"` or `"4+5"`) |
| `magic_prompt`   | string | ❌       | Custom AI instruction                    |
| `text`           | string | ❌       | Syllabus text content                    |
| `files`          | file[] | ❌       | PDF or image files (max 10)              |

```json
Response: {
  "docx_filename": "Class_9_Physics_1748291234.docx",
  "ai_data": { "mcqs": [...], "short_qs": [...], "long_qs": [...] }
}
```

<br>

### `POST /refine-test`

Updates an existing test using an AI command. Accepts same params as `/process-test` plus:

| Parameter       | Type   | Required | Description                      |
| --------------- | ------ | -------- | -------------------------------- |
| `ai_data_json`  | string | ✅       | Current test data as JSON string |
| `refine_prompt` | string | ✅       | Instruction for the AI editor    |

<br>

### `GET /get-file/{filename}`

Downloads the generated `.docx` file.

```
GET /get-file/Class_9_Physics_1748291234.docx
→ Returns: application/vnd.openxmlformats-officedocument.wordprocessingml.document
```
